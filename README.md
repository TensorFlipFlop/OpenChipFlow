# OpenChipFlow

> **⚠️ Notice: Under Active Development**
> OpenChipFlow is currently undergoing heavy development. Due to AI token constraints, progress is deliberate, and the toolchain is not yet ready for general use. We are targeting **mid-April** for a stable open-source release to the community.

OpenChipFlow is an advanced, AI-driven open-source chip design workflow. It is designed to automate and orchestrate the entire IC design lifecycle—from reading specifications to pushing automated RTL and testbench verification code—using LLMs in a controlled, systematic manner.

## Key Operational Modes

OpenChipFlow currently operates in three primary modes to accommodate different stages of AI-assisted hardware development:

### 1. Spec Flow
A fully automated, end-to-end pipeline driven purely by a hardware specification (`spec.md`). The flow advances through the following stages:
- **Precheck**: Performs environment and dependency validation.
- **Plan**: Analyzes the specification to generate architectural plans, detailed design documents, and verification plans.
- **Implement**: Materializes the planned documents into actual RTL code and Testbench (TB) components.
- **Verify**: Executes the verification testsuite according to the verification plan and analyzes the simulation results.
- **Regress**: Runs comprehensive test regressions to ensure robustness and generates analytical reports on coverage and edge cases.

### 2. Handoff Intake
A compliance and review mode for artifacts generated upstream. Driven by a "handoff" directory (which can consist of manually drafted plans or code generated by an external AI session), this mode performs strict compliance verification:
- Evaluates the layout, file contracts, and semantics of the submitted code and planning documents.
- Automatically generates targeted feedback and repair instructions (tailored AI prompts) for any defects or missing pieces.
- Iterates repeatedly via repair prompts until the handoff is officially certified as complete and verified.

### 3. Verify-Ready Handoff
A downstream execution mode that directly bridges from the `Handoff Intake` mode. Once a handoff folder successfully passes the intake checks, this mode takes that certified package and pushes it through the rest of the flow:
- Bypasses the initial planning phase, trusting the certified handoff context.
- Directly executes the implementation, verification, and regression loops on the provided codebase.
